1.2 Xây dựng thương hiệu
  • Tên thương hiệu: tiếng Anh hoặc tiếng Việt, ngắn gọn, dễ nhớ.
  • Logo: thiết kế bằng Figma / Canva, đơn giản nhưng nhận diện được.
  • Bộ màu: tối thiểu 2 màu chủ đạo + 1 màu nhấn (accent).
  • Font chữ: 1 font heading + 1 font body, áp dụng nhất quán toàn site.
  • Tagline / slogan: 1 câu ngắn mô tả giá trị thương hiệu.
1.3 Thiết kế Figma (bắt buộc)
  • Wireframe cho tất cả trang thành phần (khung bố cục, chưa cần màu sắc).
  • Mockup màu đầy đủ cho tất cả trang, áp dụng bộ màu và font thương hiệu.
  • Responsive: thiết kế cho 3 breakpoint – Mobile (375px) · Tablet (768px) · Desktop (1280px).
  • Component library: button, card, input field, navbar, footer, badge… đồng nhất toàn bộ.
  • Nộp link Figma (view-only) trong file README.md của repository.
2
Xây dựng Website với NextJS và Tailwind CSS
2.1 Yêu cầu kỹ thuật
  • Framework: NextJS 14+ với App Router.
  • Styling: Tailwind CSS. Cho phép dùng thêm shadcn/ui, Headless UI hoặc Radix UI.
  • Animation: Framer Motion hoặc Tailwind transitions – tối thiểu 3 hiệu ứng chuyển động.
  • State management: Zustand hoặc React Context (giỏ hàng, auth state…).
  • Dữ liệu: JSON tĩnh hoặc mock API (json-server, mockAPI.io). Không bắt buộc backend thật.
  • Hình ảnh: dùng next/image, ảnh chất lượng cao từ Unsplash / Pexels.
  • Code quality: ESLint không lỗi, cấu trúc thư mục rõ ràng (components/, app/, lib/, hooks/).
2.2 Các trang thành phần bắt buộc (tối thiểu 10 trang)
# Tên trang Yêu cầu chức năng
1 Trang chủ (Home) Banner / hero section, sản phẩm nổi bật, danh mục, khuyến mãi.
2 Danh sách sản phẩm Lọc theo danh mục / giá / đánh giá, sắp xếp, phân trang hoặc infinite scroll.
3 Chi tiết sản phẩm Gallery ảnh, mô tả, thông số kỹ thuật, đánh giá người dùng, sản phẩm liên quan.
4 Giỏ hàng Thêm / xoá / cập nhật số lượng, tính tổng tiền, áp mã giảm giá.
5 Thanh toán (Checkout) Form địa chỉ giao hàng, chọn phương thức thanh toán, tóm tắt đơn hàng.
6 Đăng nhập / Đăng ký Form validation, thông báo lỗi rõ ràng, tính năng remember me.
7 Tìm kiếm Tìm kiếm theo từ khoá, highlight kết quả, gợi ý tìm kiếm (autocomplete).
8 Liên hệ (Contact) Form liên hệ có validation, bản đồ nhúng (Google Maps hoặc Leaflet).
9 About / Giới thiệu nhóm Ảnh + thông tin từng thành viên, MSSV, vai trò, link GitHub cá nhân.
10 404 Not Found Trang lỗi thân thiện, có nút điều hướng quay lại trang chủ.
11+ Trang đặc trưng theo chủ đề Blog / FAQ / Chính sách / Dashboard người dùng / Lịch sử đơn hàng… (tùy chọn, cộng điểm)
2.3 Yêu cầu UX / UI
  • Header: logo, navigation menu, icon giỏ hàng (hiển thị số lượng), nút đăng nhập.
  • Footer: link các trang, thông tin liên hệ, icon mạng xã hội.
  • Loading state: skeleton loader hoặc spinner khi fetch dữ liệu.
  • Toast notification: thông báo khi thêm vào giỏ hàng, gửi form thành công / thất bại.
  • Breadcrumb: điều hướng tại trang danh sách và chi tiết sản phẩm.
  • Dark mode (tùy chọn nâng cao – cộng điểm thưởng).
3
Deploy và Quản lý Repository GitHub
3.1 Cấu hình GitHub Repository
  • Tên repo: format nhomXX_tenchude (ví dụ: nhom05_cosmetics) – để chế độ Public.
  • Mỗi thành viên có ít nhất 5 commits có ý nghĩa (không tính merge commit).
  • Sử dụng GitHub Issues để quản lý công việc – ít nhất 10 issues đã đóng.
  • Có ít nhất 1 Pull Request được review và merge trong quá trình phát triển.
3.2 Deploy lên GitHub Pages (bắt buộc để giảng viên chấm)
  1. Thêm output: 'export' và basePath: '/ten-repo' vào next.config.js. Thêm images: { unoptimized: true }.
  2. Tạo file .github/workflows/deploy.yml với GitHub Actions: build → upload artifact → deploy to Pages.
  3. Vào Settings → Pages → Source: chọn GitHub Actions.
  4. Sau deploy, URL có dạng: https://username.github.io/ten-repo/
  5. Dán URL này vào README.md và file báo cáo Word.
⚠️
Lưu ý kỹ thuật: NextJS App Router với output: 'export' không hỗ trợ API Routes và dynamic routes cần server-side rendering. Chuyển toàn bộ fetch dữ liệu sang client-side hoặc dùng Static Generation (generateStaticParams).
3.3 Nội dung README.md bắt buộc
  • Tên nhóm, danh sách thành viên (Họ tên đầy đủ + MSSV + vai trò trong dự án).
  • Mô tả dự án: chủ đề, tên thương hiệu, tính năng nổi bật.
  • 🔗 Link Figma (view-only)  ·  🔗 Link GitHub Pages  ·  🔗 Link video YouTube (nếu có).
  • Hướng dẫn chạy local: npm install → npm run dev.
  • Ảnh chụp màn hình giao diện (tối thiểu 3 ảnh các trang chính).
4
Báo cáo và Thuyết trình
4.1 Cấu trúc báo cáo Word
  1. Lời mở đầu – giới thiệu nhóm và bảng phân công công việc.
  2. Giới thiệu chủ đề: lý do chọn, đối tượng người dùng mục tiêu, phân tích đối thủ cạnh tranh.
  3. Thiết kế Figma: quyết định về màu sắc / font / layout, ảnh chụp màn hình Figma.
  4. Kiến trúc kỹ thuật: sơ đồ thư mục, danh sách component chính, luồng dữ liệu.
  5. Tính năng nổi bật: mô tả và giải thích code các tính năng phức tạp nhất.
  6. Hướng dẫn deploy: các bước cấu hình GitHub Actions, các lỗi gặp và cách xử lý.
  7. Kết luận và hướng phát triển tiếp theo.
  8. Tài liệu tham khảo (≥10 nguồn, theo chuẩn APA).
4.2 Buổi vấn đáp
  • Demo live trên GitHub Pages – không được demo trên localhost.
  • Mỗi thành viên trình bày phần mình phụ trách và giải thích code tương ứng.
  • Giảng viên có thể yêu cầu sửa đổi nhỏ trực tiếp trên code trong buổi vấn đáp.
  • Thời gian mỗi nhóm: 15–20 phút.
4.3 Video YouTube (không bắt buộc nếu vấn đáp offline)
  • Thời lượng: 5–10 phút. Chế độ: Public.
  • Nội dung: demo các tính năng chính, giải thích kỹ thuật và phân công nhóm.
  • Tất cả thành viên phải xuất hiện trong video và có phần thuyết trình của riêng mình.
